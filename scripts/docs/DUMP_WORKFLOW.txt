Torrent-first workflow (long-range Reddit)
==========================================

1) Acquire data
   Monthly dumps are usually named RC_YYYY-MM.zst (comments) and RS_YYYY-MM.zst
   (submissions), newline-delimited JSON inside zstd. Academic Torrents hosts
   rolling "Reddit comments/submissions" collections, e.g. search that site for
   https://academictorrents.com/ and the newest monthly bundle. Download only
   the months you will process — each file is tens of GB uncompressed-equivalent.

   Command-line download (Academic Torrents docs recommend aria2c):
   - macOS: brew install aria2
   - Use the .torrent URL from YOUR dataset’s page on academictorrents.com (the
     infohash in examples below is illustrative only — copy the real link).
   - Download into this project’s dumps/ folder:

     aria2c --seed-time=0 --seed-ratio=0 \
       --max-overall-download-limit=10M --file-allocation=none \
       --dir="FULL_PATH_TO_PROJECT/dumps" \
       "https://academictorrents.com/download/<INFOHASH>.torrent"

   The full “2005–2025” bundle can be ~3.8 TB — do NOT pull everything unless
   you mean to. List files in the torrent, then select indices:

     aria2c --list-file https://academictorrents.com/download/<INFOHASH>.torrent

     aria2c --seed-time=0 --seed-ratio=0 --file-allocation=none \
       --dir=".../dumps" --select-file=1,2,3 \
       "https://academictorrents.com/download/<INFOHASH>.torrent"

   (Replace 1,2,3 with the line numbers for the RC_/RS_ months you want.)

   Pure-Python alternative (installs via pip):

     pip install academictorrents
     at-get <INFOHASH>

   Point output to dumps/ per that tool’s docs if it supports a --dir / cwd.

2) Install
   python3 -m venv .venv && source .venv/bin/activate
   pip install -r requirements.txt

2a) Month-by-month + parallel workers (saves disk: one month download → ingest → delete staging)
   Stop any single huge aria2 job first (kill that process).
     python pipeline_months.py --workers 3 --start 2020-01 --end 2025-12
   Uses dumps/.staging/YYYY-MM/ per job; raw .zst removed after ingest. Parquet merge is file-locked.

2b) Managed daemon (recommended — polls forever, ingests each new .zst once)
   ./start_manager.sh
   Log: logs/manager.log   PID: logs/manager.pid   manifest: state/processed_dumps.txt
   Stops duplicate work: only unseen dump files are ingested; merges into data/reddit_boston_housing.parquet
   If .env has Reddit API keys, runs collect_reddit.py every 2h when no dumps are present.

2c) One-shot overnight (waits up to 12h for .zst files, then ingests all globs once)
   ./run_overnight.sh
   Logs: logs/ingest_*.log  —  monitor:  tail -f logs/ingest_*.log
   Env: DUMPS=/path/to/zst  WAIT_MAX_MINUTES=720  FLUSH_MATCHES=250000

3) Ingest (filter to Boston subs + housing/commute keywords — see config.yaml)
   Example — one month, both comments and submissions (streaming chunks to limit RAM):
     python ingest_reddit_dump.py --glob "dumps/RC_2020-01.zst" --glob "dumps/RS_2020-01.zst" --fresh --flush-matches 200000

   Add more months (merge into same Parquet, dedupe by id):
     python ingest_reddit_dump.py --glob "dumps/RC_2020-02.zst" --glob "dumps/RS_2020-02.zst" --flush-matches 200000

   Trial run (first N lines of each file):
     python ingest_reddit_dump.py --glob "dumps/RC_2020-01.zst" --max-lines 200000

   Widen net (subreddit only, then filter later in pandas):
     python ingest_reddit_dump.py --glob "dumps/*.zst" --no-keyword-filter --max-matched-rows 500000 --flush-matches 200000

4) Embed + t-SNE + Plotly
     python embed_tsne_viz.py --in data/reddit_boston_housing.parquet

   Multiple yearly Parquet shards (if you split outputs):
     python embed_tsne_viz.py --in data/part2015.parquet data/part2016.parquet

5) Optional live top-up after history is built
     python collect_reddit.py
